# **Глоссарий терминов Python**

*   **Python:** Высокоуровневый язык программирования общего назначения, разработанный Гвидо ван Россумом.
*   **PEP (Python Enhancement Proposal):** Документ, предоставляющий информацию сообществу Python, или описывающий новую функцию для Python, ее процессы или окружение.
*   **GIL (Global Interpreter Lock):** Глобальная блокировка интерпретатора, позволяющая одновременно выполнять только один поток Python в одном процессе.
*   **List comprehension (Генератор списка):** Компактный способ создания новых списков из существующих итерируемых объектов.
*   **Dict comprehension (Генератор словаря):** Компактный способ создания словарей.
*   **Tuple (Кортеж):** Упорядоченная, неизменяемая последовательность элементов.
*   **List (Список):** Упорядоченная, изменяемая последовательность элементов.
*   **Dictionary (Словарь):** Структура данных, хранящая пары ключ-значение.
*   **Function (Функция):** Блок организованного, повторно используемого кода, который выполняет определенную задачу.
*   **Class (Класс):** Шаблон для создания объектов, определяющий атрибуты и методы.
*   **Object (Объект):** Экземпляр класса.
*   **Module (Модуль):** Файл, содержащий определения и инструкции Python.
*   **Package (Пакет):** Каталог, содержащий модули Python.
*   **Decorator (Декоратор):** Функция, которая принимает другую функцию в качестве входных данных и расширяет ее функциональность.
*   **Virtual environment (Виртуальное окружение):** Изолированное окружение для проектов Python, позволяющее управлять зависимостями.
*   **pip:** Менеджер пакетов для Python, используемый для установки и управления пакетами.
*   **venv:** Модуль Python для создания и управления виртуальными окружениями.
*   **Interpreter (Интерпретатор):** Программа, которая выполняет код Python.
*   **Syntax (Синтаксис):** Набор правил, определяющих структуру языка Python.
*   **Semantics (Семантика):** Значение, которое придает синтаксис.
*   **Exception (Исключение):** Событие, которое прерывает нормальный поток выполнения программы.
*   **try-except:** Конструкция для обработки исключений.
*   **if-else:** Конструкция для условного выполнения кода.
*   **for loop (Цикл for):** Конструкция для итерации по последовательности.
*   **while loop (Цикл while):** Конструкция для циклического выполнения кода до тех пор, пока условие истинно.
*   **Variable (Переменная):** Именованная область памяти, которая хранит значение.
*   **Data type (Тип данных):** Классификация данных (например, integer, string, boolean).
*   **String (Строка):** Последовательность символов.
*   **Integer (Целое число):** Целое число.
*   **Float (Число с плавающей запятой):** Вещественное число.
*   **Boolean (Логический тип):** Тип данных, представляющий истину (True) или ложь (False).
*   **API (Application Programming Interface):** Интерфейс программирования приложений.
*   **JSON (JavaScript Object Notation):** Формат обмена данными.
*   **ORM (Object-Relational Mapper):** Инструмент для работы с базами данных.
*   **SQLModel:** Библиотека для работы с базами данных на основе Pydantic.
*   **FastAPI:** Современный, быстрый (высокопроизводительный), готовый к использованию веб-фреймворк для создания API с Python 3.7+ на основе аннотаций типов.
*   **Docker:** Платформа для разработки, развертывания и запуска приложений в контейнерах.
*   **Dockerfile:** Текстовый файл, содержащий инструкции для сборки образа Docker.
*   **Docker Compose:** Инструмент для определения и запуска многоконтейнерных приложений Docker.
*   **Container (Контейнер):** Легкий, автономный, исполняемый пакет программного обеспечения, включающий все необходимое для его запуска: код, среду выполнения, системные инструменты, системные библиотеки и настройки.
*   **Image (Образ):** Шаблон для создания контейнеров, включающий в себя все необходимое для запуска приложения.
*   **Volume (Том):** Механизм для сохранения данных, генерируемых и используемых контейнерами Docker.
*   **Environment variables (Переменные окружения):** Динамические значения, которые влияют на то, как работают запущенные процессы в операционной системе.
*   **Dependency (Зависимость):** Компонент, необходимый для работы другого компонента.
