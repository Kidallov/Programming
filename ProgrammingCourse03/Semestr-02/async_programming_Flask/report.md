# Отчет по самостоятельной работе
**Дисциплина:** Программирование на Python  
**Тема:** Асинхронное программирование на Python с применением Flask и asyncio  
**Используемые технологии:** Flask, asyncio, aiohttp, Bootstrap 5, Jinja2  

---

## 1. Постановка задачи и цели работы

**Цель работы:** Освоение механизмов асинхронного программирования в экосистеме Python с интеграцией веб-фреймворка Flask и библиотек `asyncio` и `aiohttp`. В качестве источника данных используется внешний REST API сервис `JSONPlaceholder`.

**Основные задачи:**
1. Разработка единой веб-формы поиска с четырьмя необязательными фильтрационными полями (`username`, `email`, `title`, `body`).
2. Реализация конкурентных неблокирующих сетевых HTTP-запросов к эндпоинтам `/users` и `/posts` удаленного API.
3. Оптимизация агрегации данных: параллельный запрос авторов для каждого найденного поста с применением конструкции `asyncio.gather`.
4. Обеспечение отказоустойчивости приложения при обработке сетевых аномалий (ошибки API, пустые ответы, таймауты соединения).
5. Создание адаптивного и отзывчивого пользовательского интерфейса с использованием CSS-фреймворка Bootstrap 5.

---

## 2. Теоретическая справка и логика фильтрации

Сервис **JSONPlaceholder** предоставляет фейковые REST-ресурсы для тестирования. В рамках данной работы сетевой обмен оптимизирован следующим образом:
* Вместо множества мелких запросов с query-параметрами приложение асинхронно скачивает полные списки ресурсов, если заполнено хотя бы одно профильное поле формы.
* Фильтрация (поиск подстроки без учета регистра) выполняется на стороне клиента (сервера Flask). Это наглядно демонстрирует асинхронную постобработку массивов данных.
* **Решение проблемы N+1 запросов:** При выводе постов требуется отобразить имя автора по его `userId`. Запрос информации о пользователях через `/users/{id}` для каждого поста по отдельности привел бы к последовательному замедлению работы. Внедрение `asyncio.gather` позволяет мультиплексировать сетевые сессии и запрашивать данные обо всех авторах конкурентно (одновременно).

## 3. Спецификация полей формы поиска

| Поле формы | Описание алгоритма поиска | Ресурс REST API |
| :--- | :--- | :--- |
| **Имя пользователя** (`username`) | Поиск пользователей, чей `username` содержит подстроку (регистронезависимо) | `/users` |
| **Email пользователя** | Поиск пользователей, чей `email` содержит подстроку (регистронезависимо) | `/users` |
| **Заголовок поста** (`title`) | Поиск постов, у которых `title` содержит подстроку (регистронезависимо) | `/posts` |
| **Текст поста** (`body`) | Поиск постов, у которых `body` содержит подстроку (регистронезависимо) | `/posts` |
