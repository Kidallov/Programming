# Отчет по лабораторной работе №1: «CI/CD с использованием SourceCraft на примере Hugo»

## Цель работы

Целью данной лабораторной работы является изучение принципов CI/CD (непрерывной интеграции и доставки) с использованием платформы SourceCraft на примере статического сайта Hugo. Работа направлена на автоматизацию сборки, тестирования и развертывания проекта.

## 1. Перенос существующего проекта на SourceCraft

Исходный сайт на Hugo был размещен на GitHub:
[https://github.com/Kidallow/hugosite.github.io](https://github.com/Kidallow/hugosite.github.io)

Далее проект был перенесен на платформу SourceCraft и настроен для автоматической сборки и развертывания:
[Выполненная лабораторная работа на Sourcecraft](https://sourcecraft.dev/kidallovv/demo1803)

## 2. Настройка проверок качества Markdown

Для контроля качества исходных файлов Markdown была добавлена проверка с помощью `markdownlint`. Проверка выполняется как отдельный шаг CI/CD и предотвращает сборку проекта при наличии ошибок или несоответствий стандартам синтаксиса Markdown.

```
- name: lint-markdown                # Название шага CI/CD: проверка Markdown
  cubes:
    - name: check-syntax             # Название конкретного контейнера/куба, выполняющего проверку
      image: docker.io/tmknom/markdownlint:latest  # Используем официальный Docker-образ markdownlint последней версии
      script:
        - |
          markdownlint "themes/**/*.md"   # Запускаем проверку всех .md файлов в папке themes и её подпапках
```

## Заключение

В ходе работы был выполнен перенос статического сайта с GitHub на SourceCraft, а также реализована автоматическая проверка синтаксиса Markdown перед сборкой проекта. В результате работы были приобретены практические навыки настройки CI/CD, автоматизации тестирования и обеспечения качества кода при сборке статических сайтов.
